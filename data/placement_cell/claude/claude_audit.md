## Summary of Critical Findings

> [!CAUTION]
> **10 endpoints have NO authentication** — any unauthenticated user can create schedules, delete records, generate CVs, and modify placement data. This is an **actively exploitable security vulnerability**.

> [!WARNING]
> **~1,355 lines of dead/unreachable code** exist in views.py (lines 1549–2903). Entire functions are duplicated verbatim after `return` statements and will never execute.

> [!IMPORTANT]
> The module's **views.py is 5,811 lines** — the largest monolithic view file audited in this codebase. It contains all business logic, all DB queries, all form handling, all export logic, and all CRUD in a single file with zero separation of concerns.

### Priority Order for Refactoring

1. **P0 — Security**: Add `@login_required` + role checks to all 10 unprotected endpoints (S28–S37)
2. **P1 — Dead Code**: Remove 1,355 lines of unreachable code (S38–S41)
3. **P2 — Service Layer**: Create `services.py` + `selectors.py` (S01, S02)
4. **P3 — Duplication**: Consolidate duplicated functions (R01–R04, R07–R08)
5. **P4 — Validation**: Add input validation to all raw POST handlers (S24–S27)
6. **P5 — DRF Migration**: Create `api/views.py` + `api/urls.py` (S03, S04)
7. **P6 — Performance**: Fix N+1 query in statistics (S46)
8. **P7 — Tests**: Create comprehensive test suite (S05)
