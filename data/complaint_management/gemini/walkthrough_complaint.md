## Architectural Notes
- **Views**: Now purely handle HTTP request payloads, pass through serializers, run a service or selector, and return a dictionary mapping DRF properties.
- **Services**: Now strictly orchestrates application boundaries, handles SLA timing creation, rating algorithms, DB manipulation paths, external notification dispatches via functions without needing Request parsing.
- **Selectors**: Encapsulates common ExtraInfo lookup logic, specific location mappings, filtering by status.

All SA-# and RR-# issues tracked and successfully resolved. Test suite added ensuring system functionality.
